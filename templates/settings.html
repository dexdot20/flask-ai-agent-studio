RAG source pools

Each row maps one memory pool. Search controls whether the assistant may retrieve from that pool during normal RAG lookup. Auto inject controls whether matches from that pool may be inserted automatically into the prompt. Tool outputs come from executed tools inside chats, while saved web/fetches are remembered external results kept separately for reuse.

Chats Earlier user and assistant messages indexed from previous conversations. Useful when the assistant should recall prior discussion without relying only on the visible thread.
Tool-derived context
Tool outputs Outputs captured from executed tools inside conversation history, such as searches, fetches, or analysis steps. Use this when recent tool work should stay searchable in chat context.
Saved web/fetches Remembered web, URL, and news results stored separately from conversation history so they can be reused later without depending on the original chat turn.
Uploaded documents Manual files, notes, or other knowledge-base content added from Settings. Use this when uploaded reference material should participate in retrieval.

In practice, Search decides what the system may look up, and Auto inject decides what eligible matches may be inserted automatically. If a pool is not searchable, it will not contribute generic retrieved context even if Auto inject is checked.

Search is enabled for the selected pools only; unselected pools will not participate in generic RAG retrieval.
