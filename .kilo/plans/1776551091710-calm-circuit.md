# Plan: Add MiniMax API Support

## Context
User wants to add MiniMax API support to the Flask RAG Vision Chatbot. The API docs show MiniMax supports the Anthropic SDK format at `https://api.minimax.io/anthropic` with models like `MiniMax-M2.7`.

## Key Constraints from MiniMax API Docs
- **Temperature**: Range (0.0, 1.0] — values outside this range will return an error
- **Image/Document input**: NOT supported
- **Thinking**: Fully supported (like Claude's extended thinking)
- **Tool use**: Fully supported
- **Supported models**: MiniMax-M2.7, MiniMax-M2.7-highspeed, MiniMax-M2.5, MiniMax-M2.5-highspeed, MiniMax-M2.1, MiniMax-M2.1-highspeed, MiniMax-M2

## Implementation Steps

### 1. config.py
- Add `MINIMAX_API_KEY` env var reading
- Add `MINIMAX_PROVIDER = "minimax"` constant
- Add to `AVAILABLE_MODELS` / `AVAILABLE_MODEL_IDS` (later after model_registry update)

### 2. model_registry.py
- Add `MINIMAX_PROVIDER = "minimax"` constant
- Add MiniMax models to `BUILTIN_MODELS` list:
  - `MiniMax-M2.7` (204,800 context, ~60 tps)
  - `MiniMax-M2.7-highspeed` (~100 tps)
  - `MiniMax-M2.5` (~60 tps)
  - `MiniMax-M2.5-highspeed` (~100 tps)
  - `MiniMax-M2.1` (~60 tps)
  - `MiniMax-M2.1-highspeed` (~100 tps)
  - `MiniMax-M2` (Agentic capabilities)
- Add MiniMax client creation in `get_provider_client()`:
  - Base URL: `https://api.minimax.io/anthropic`
  - **Use Anthropic SDK** directly (not OpenAI wrapper) since MiniMax uses Anthropic API format
- Add MiniMax to `build_model_provider_policy()` with prompt cache support
- Handle `temperature` range validation: (0.0, 1.0]
- Update `DEFAULT_VISIBLE_CHAT_MODEL_ORDER` to include MiniMax models (user confirmed: include in default visible order)

### 3. agent.py
- Temperature override validation (must be within 0.0-1.0 for MiniMax)

### 4. Settings UI (frontend)
- Update model selection to include MiniMax models
- Note: Image/document inputs are NOT supported for MiniMax models

### 5. Documentation
- Update memory/PROJECT_CONTEXT.md to reflect MiniMax addition

## Temperature Handling
MiniMax requires temperature in range (0.0, 1.0]. Since the app currently uses temperature range 0.0-2.0 for other providers, we need per-provider validation:
- DeepSeek: 0.0-2.0
- OpenRouter: 0.0-2.0
- MiniMax: (0.0, 1.0] (must be > 0.0 and <= 1.0)

## Dependencies
- Add `anthropic` package to `requirements.txt`

## Notes
- MiniMax can be added alongside existing DeepSeek/OpenRouter providers
- No need to replace any existing functionality
- MiniMax API key should be optional (app should work without it)
